Profiles

Last updated: Mar 21st, 2017

Introductions

Profiles can be used to lighten loads on a single A.R.S Database.
You can also lock profiles to channels which will allow for a more dynamic feel!

Basically a Profile is a seperate A.R.S Database from your main
wich you can mantain within different channels.
You cannot link a custom profile to your main channel.

Profile Management

Creating Profiles

.create -p profile-name

Copying Profiles

Saves the current profile to a new Database.

.save -p profile-name

Deleting Profiles

.delete -p profile-name

Loading Profiles

Loads a profile to the current channel.

.load -p profile-name

Locking\Unlocking

Lock Profile

Locks a profile to a channel.

.lockprofile profile-name

Unlock Profiles

Unlocks a profile from a channel.

.unlockprofile profile-name

Check Profiles

Checks to see what profile is loaded in a channel.

.checkprofile
